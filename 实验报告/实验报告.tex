	\section{数据处理与问题定义}
	
	\subsection{数据来源与范围}
	本次实验使用 A 股市场全部上市公司的日频基础量价数据，时间范围为 2016 年 1 月至 2026 年 6 月。数据包含以下字段：
	\begin{itemize}
		\item \texttt{ts\_code}：股票代码
		\item \texttt{trade\_date}：交易日期
		\item \texttt{open, high, low, close}：开盘价、最高价、最低价、收盘价
		\item \texttt{vol}：成交量
		\item \texttt{amount}：成交额
		\item \texttt{pct\_chg}：涨跌幅
		\item \texttt{vwap}：均价
	\end{itemize}
	此外，可选使用基本面、资金流向及新闻文本数据，本报告基于量价数据完成基础建模，并在 LSTM 模型中进行了特征扩展对比实验。
	
	\subsection{股票池筛选}
	为与模拟交易比赛规则保持一致，我们进行以下过滤：
	\begin{enumerate}
		\item 剔除 ST 股：通过读取 \texttt{stock\_st/} 目录下的所有 csv 文件，合并历年 ST 股票列表，过滤掉任一时期被标记为 ST 的股票。
		\item 剔除北交所股票：依据 \texttt{basic.csv} 中的 \texttt{market} 字段，排除市场类别为“北交所”的股票。
		\item 暂不限制创业板/科创板，保留全部主板、创业板、科创板股票（约 5000 只）。
	\end{enumerate}
	上述过滤在数据加载阶段完成，保证了训练样本与比赛交易标的的一致性。
	
	\subsection{数据加载与缺失值处理}
	数据以每日单个 CSV 文件形式存储（文件名格式 \texttt{YYYYMMDD.csv}）。我们首先读取交易日历 \texttt{trade\_cal.csv}，筛选出 \texttt{is\_open==1} 的交易日，然后按日期顺序逐日加载并合并为面板数据。缺失值处理遵循以下原则：
	\begin{itemize}
		\item 对于特征列（如收盘价、成交量等），在每只股票的时间序列内使用前向填充（\texttt{ffill}），即用最近一个有效值填补缺失值。
		\item 填充后仍存在缺失值的样本（如股票上市初期无前序数据）将被删除，避免引入噪声。
	\end{itemize}
	上述处理均在分组（按股票）且按时间排序后执行，未使用任何未来信息。

	\subsection{标签定义}
	采用回归任务设定：预测未来 \(n\) 个交易日的收益率（\(n=1\)，即 T+1 收益率）。对于每个交易日 \(t\) 的股票，标签计算为：
	\[
	\text{label}_t = \frac{\text{close}_{t+n} - \text{close}_t}{\text{close}_t}
	\]
	
	\subsection{滑动窗口样本构造}
	为了使模型能够捕捉时间依赖关系，我们将每只股票的历史序列切分为固定长度的输入窗口和目标标签。窗口长度均为 \(L=20\) 个交易日（Transformer）或 \(L=30\) 个交易日（LSTM）。对于股票 \(s\) 在交易日 \(d\) 的样本：
	\begin{itemize}
		\item 输入特征：\(\mathbf{X} = [\mathbf{f}_{d-L}, \mathbf{f}_{d-L+1}, \ldots, \mathbf{f}_{d-1}]\)，维度 \(L \times F\)。
		\item 标签：根据模型定义取对应未来收益率或K线变化率。
	\end{itemize}
	
	\subsection{数据集划分}
	为保证时间序列的因果性，我们严格按照时间顺序划分训练集和验证集，禁止随机打乱。划分方案如下：
	\begin{itemize}
		\item 训练集：2016年1月1日 – 2024年12月31日
		\item 验证集：2025年1月1日 – 2025年12月31日
	\end{itemize}
	
	\subsection{标准化策略与避免数据泄露}
	金融时间序列不同特征的量纲差异较大，需要标准化。我们的做法如下：
	\begin{enumerate}
		\item 仅使用训练集中所有样本的输入窗口内全部特征值计算均值和标准差。
		\item 利用训练集的统计量对训练集本身进行标准化（\(z = (x - \mu)/\sigma\)）。
		\item 对验证集以及后续的末日推理数据，使用相同的 \(\mu, \sigma\) 进行变换，不再重新拟合。
	\end{enumerate}

	\subsection{LSTM 模型}
	
	\subsubsection{设计理念}
	传统多任务学习将三个预测目标视为独立任务，忽略了它们之间的内在因果关系。本模型提出\textbf{级联关联架构}，核心思想是三个分支级联关联——前一个分支的预测结果作为后一个分支的额外输入。因果关系链：
	\begin{itemize}
		\item OP分支：纯从历史特征预测开盘价跳空方向。
		\item LP分支：在知道开盘价后，预测最低价（开盘后能跌多深）。
		\item HP分支：在知道开盘价和最低价后，预测最高价（全天反弹高度）。
	\end{itemize}
	
	\subsubsection{AssociatedResidualNet 架构}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.9\textwidth]{ChatGPT_Image_22_03_16.png}
		\caption{多值关联残差网络架构}
		\label{fig:model_arch}
	\end{figure}
	
	架构流程：
	\begin{enumerate}
		\item OP分支：输入原始特征 \(x\)，经过 ResidualLSTMBlock，输出 \(pred\_op\)。
		\item LP分支：输入 \([x, pred\_op]\) 级联，经过 ResidualLSTMBlock，输出 \(pred\_lp\)。
		\item HP分支：输入 \([x, pred\_op, pred\_lp]\) 级联，经过 ResidualLSTMBlock，输出 \(pred\_hp\)。
	\end{enumerate}
	
	每个 ResidualLSTMBlock 由 LSTM 层、Dropout、LayerNorm 和残差连接组成：
	\[
	output = LayerNorm(LSTM(x) + Dropout(x))
	\]
	当维度不匹配时通过 Linear 投影层调整。

	\subsubsection{损失函数}
	采用复合损失：
	\[
	Loss_{total} = Loss_{regression} + \lambda \cdot Loss_{ranking}
	\]
	其中 \(Loss_{regression}\) 为联合方向回归损失（MSE 加方向性惩罚），\(Loss_{ranking}\) 为 ListNet 截面排序损失，\(\lambda = 0.1\)。

	\subsection{Transformer 模型回测}
	
	\subsubsection{回测设置}
	基于验证集（2025年1月至12月）导出的预测分数，采用 \texttt{backtest\_score\_weighted.py} 进行历史回测。策略核心：
	\begin{enumerate}
		\item 每个交易日使用上一交易日的预测分数（滞后1天）。
		\item 选出预测分数最高的30只股票作为候选池，再取前10只持仓。
		\item 权重与预测分数成正比（池内平移后归一化）。
		\item 每日早盘清空所有可卖仓位，按权重满仓买入，价格采用当日收盘价。
		\item 交易费用：印花税（卖出0.1\%）、过户费（沪市万六，最低1元）、券商佣金（万三，单笔最低5元），回测中佣金基点设为0。
		\item 初始资金100万元，最低交易单位100股。
	\end{enumerate}

	\subsection{LSTM 模型模拟交易表现}
	实际上我并不会每天买卖5只股票，而是根据模型预测得分动态调整持仓数量。首先是我不能每天九点起床，可能10点以后才开始交易。如果发现模型给我推荐的股票开盘后暴涨，我肯定不会买（毕竟我的模型预测和开盘 价最高价高度相关），卖出是也同理。所以实际上我每天可能只买卖两三只，不过这都是模型给我推荐的，完全在他的预测之内。

	\subsection{经验与反思}
	从两个模型的模拟交易中我们总结了以下几点：
	\begin{enumerate}
		\item \textbf{数据质量与执行偏差}：盘中实时价格与盘后收盘价的差异可能导致指令执行偏差；周末无交易导致周一预测难度较大，两次较大亏损均发生在周一。
		\item \textbf{策略稳健性}：Transformer 模型的高换手满仓策略在回测中表现不佳，而 LSTM 模型的适度调仓（每日最多调仓3只）和更低换手显著提升了净收益。LSTM 模型在非理想执行条件下仍能盈利，说明其预测信号更具稳健性。
		\item \textbf{特征选择}：复杂的模型和更大的数据并未带来更好效果；基本面、资金流等扩展特征在当前数据集上提升有限，简洁的特征组合有助于泛化。
		\item \textbf{市场理解}：股票可分为进攻型（高波动）和防守型（低波动），模型对进攻型股票的预测更为有效；模拟交易最后一天买入格力电器等防守型股票以求稳。
		\item \textbf{模型迭代闭环}：比赛期间无法重训模型，未来可引入在线学习或每日增量更新。
		\item \textbf{团队协作}：分工明确提高了效率，每日复盘及时调整指令生成脚本。
	\end{enumerate}
